Attack-chain correlation · how the findings connect

Individual findings are leads. AgentAutopsy correlates them into per-identity kill-chain stories — initial access, persistence, privilege escalation, collection, exfiltration, impact — and scores the whole incident. This is the narrative an examiner writes, generated automatically.

Memory poisoning · the time gap runtime tools miss (OWASP ASI06)

Memory poisoning plants an instruction that sits dormant for days or weeks, then executes when the agent retrieves it as trusted context. The write and the harmful act are separated in time — so runtime guardrails never connect them. A retrospective reconstruction is the only way to see it. Matching survives paraphrase and re-chunking (SimHash), and poison spread across agents is traced.

Detection catalog · versioned rules — the agent TTP knowledge base

Every detection is a discrete, semantically versioned rule mapped to OWASP-Agentic / MITRE ATT&CK·ATLAS, scored against a labeled benchmark on every change. Findings are confidence-rated leads; the examiner concludes.

Why this matters

    

      
The question nobody can answer

        
When an AI agent misbehaves — through compromise, prompt injection, or a cascade across
          delegated sub-agents — security teams are left asking what did it actually do?
          Most can't say. The evidence is scattered across model logs, cloud trails, and SaaS audit
          records that were never built to be read together.

      
Evidence-grade, by construction

        
Every raw record is hashed on ingest and committed to an append-only, hash-chained ledger.
          Edits, deletions, and re-hash attempts all break the chain. The reconstruction is
          deterministic and the findings carry explicit confidence — leads for an examiner, never
          unverified conclusions.
